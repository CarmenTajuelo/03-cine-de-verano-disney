Paso 2: Crear la función de lectura básica

El siguiente paso es crear una función que use esa URL para hacer la petición. Te propongo que escribas esta línea:

¿Por qué esta línea?

const getAllFilms =: Creamos una función con un nombre descriptivo que indica que va a obtener TODAS las películas.

async: La ponemos como asíncrona porque las peticiones HTTP tardan tiempo y no queremos que la aplicación se "congele" esperando la respuesta.

() =>: Es la sintaxis de función flecha (arrow function) de JavaScript.

¿Qué es una arrow function?

Una arrow function (función flecha) es una forma más corta de escribir funciones en JavaScript. Se llama así por la "flecha" => que usa.

Forma tradicional:

function getAllFilms() {
          // código aquí
        }

Forma con arrow function:

const getAllFilms = () => {
          // código aquí
        }

¿Por qué usar arrow functions?

  1. Más corta: Menos código para escribir
  2. Moderna: Es la forma actual que se usa en JavaScript
  3. Común en servicios: Prácticamente todos los servicios modernos las usan

En tu caso específico:

const getAllFilms = async () => {

Esto significa: "Crea una función llamada getAllFilms que es asíncrona y no recibe parámetros"

El siguiente paso es escribir la línea que hace la petición al servidor. Añade esta línea dentro de la función:

await = Espera a que el servidor responda antes de continuar

const response = Guarda la respuesta del servidor en una variable llamada "response"

Paso 4: Convertir la respuesta a JSON

El siguiente paso es añadir esta línea:

const data = await response.json();

¿Qué hace esta línea?

response.json() = Convierte la respuesta del servidor a formato JSON (que es como JavaScript puede entender los datos)

await = Espera a que termine la conversión

const data = Guarda los datos ya convertidos en una variable llamada "data"

¿Por qué necesitamos esto?

Cuando el servidor te envía los datos, vienen en formato "crudo" (como un mensaje cifrado). La función .json() los "traduce" a un formato que JavaScript puede usar.

En palabras simples: "Toma la respuesta del servidor y conviértela en datos que pueda usar"

Paso 5: Devolver los datos

El siguiente paso es añadir esta línea:

return data;

¿Qué hace esta línea?

return = Devuelve/entrega los datos a quien llamó a la función

data = Los datos que obtuvimos del servidor en el paso anterior

¿Por qué necesitamos return?

Imagina que tu función es como un mensajero:

  • Va a buscar las películas (fetch)
  • Las traduce para que las entiendas (json)
  • Te las entrega (return)

Sin return, sería como si el mensajero fuera a buscar tus películas pero nunca te las diera.

En palabras simples: "Entrega los datos de las películas a quien use esta función"
